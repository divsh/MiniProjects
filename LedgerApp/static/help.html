1. Overview

Ledger is a web app for tracking personal lending and borrowing transactions. All data is stored directly in a Google Spreadsheet. The app reads and writes to the sheet via Google Apps Script — no separate server or database is needed.

Access is restricted to approved Google accounts configured by the administrator.

↑ Top

2. Signing In

  1. Open the app URL in your browser.
  2. Click the Sign in with Google button.
  3. Choose your Google account.
  4. If your account is not on the approved list, access is denied.

Your session stays active for the current browser tab. Closing the tab signs you out. Use the Sign out button in the top-right header to sign out manually.

4. Selecting an Account

The Account field in the Add Transaction form controls everything on the page:

  • Click or start typing to see a filtered list of existing accounts.
  • Select All Accounts to see a summary of all accounts and their balances.
  • Select a specific account to load its balance and transaction history.

The last selected account is remembered when you refresh the page.

Hints below the field

New ledger will be created: "alice (NZD)" — the name you typed is valid and a new account will be created on save.

Include a currency code, e.g. "johnSmith (NZD)" — the name is missing a currency code.

No hint — the name matches an existing account.

5. Account Name Format

Account names must end with a three-letter currency code in parentheses:

Valid examplesInvalid examples
johnSmith (NZD)John Smith — missing currency
aliceBrown (AUD)johnSmith NZD — not in parentheses
raj (USD)

Rules for new account names

  • The name part is stored in camelCase (no spaces).
    Example: john smith (NZD) → stored as johnSmith (NZD)
  • The currency code is always stored in UPPERCASE.
    Example: alice (nzd) → stored as alice (NZD)
  • Matching is case-insensitive — typing JOHNSMITH (NZD) finds the existing johnSmith (NZD) account.

6. All Accounts View

Select All Accounts from the account dropdown to see a summary across all accounts.

Balance Card (top)

  • Account name shows All Accounts.
  • Balance shows the total outstanding per currency (e.g. NZD 3,200.00 and AUD (500.00) as separate lines).
  • Each currency total is colour-coded.

Accounts Summary Table

  • Lists every account with its current outstanding balance.
  • Balances are colour-coded: green (positive), red (negative), grey (zero).

10. Mobile Use

The app is optimised for mobile screens (640 px and below):

  • All form fields stack vertically for easy tapping.
  • The email address is hidden in the header to save space.
  • The transaction table switches to a card-per-row layout — each transaction is shown as a small card with labelled fields, avoiding horizontal scrolling.

11. Data & Storage

  • All transactions are saved directly to the Google Spreadsheet.
  • Column H (Account Balance) contains spreadsheet formulas — do not manually clear or overwrite column H.
  • The app caches all account and transaction data when it loads. After saving a transaction the cache is automatically refreshed.
  • No data is stored on any third-party server.

12. Administrator Notes

Adding or removing approved users

  1. Open the Apps Script project linked to the spreadsheet.
  2. Go to Project Settings → Script Properties.
  3. Edit the ALLOWED_EMAILS property (comma-separated Gmail addresses).
  4. No redeployment needed — the change takes effect immediately.

Required Script Properties

SPREADSHEET_ID  — ID from the Google Sheets URL

SHEET_NAME  — Tab name, e.g. Common Ledger

GOOGLE_CLIENT_ID  — OAuth 2.0 Client ID from Google Cloud Console

ALLOWED_EMAILS  — Comma-separated approved Gmail addresses

Updating the app code (Code.gs)

  1. Paste updated code into the Apps Script editor and save.
  2. Deploy → Manage deployments → Edit (pencil) → New version → Deploy.
  3. The /exec URL stays the same — no need to update app.js.
